# Финализация спринта (проект see_elecctrical_copy)

Завершить спринт: подсчитать задачи, записать итоговый отчёт, выставить статус, синкнуть ROADMAP/индекс, предложить уроки в memory и коммит/PR.

Конвенции: спринты `docs/sprints/S{N}-{slug}.md`, индекс `docs/SPRINTS.md`, карта `docs/ROADMAP.md`, memory `C:\Users\Leonid\.claude\projects\d--Projects-see-elecctrical-copy\memory\`, ветки от `main`. Без шины/`/dispatch`/forgejo/BACKLOG.

## Шаги

### 1. Найти файл
`$ARGUMENTS` → Glob `docs/sprints/S*.md`, выбрать по номеру. Без аргумента — найти активный (`🚧|В работе`). Прочитать целиком.

### 2. Подсчитать задачи
```bash
grep -c '\- \[x\]' <файл>
grep -c '\- \[ \]' <файл>
```
Показать: ✅ X · ⬜ Y · 📈 Z%.

### 3. Если есть невыполненные (Y>0)
AskUserQuestion: «Финализировать с невыполненными (пометить незавершёнными)?» / «Отменить — сначала добить». Если Y=0 — без вопроса.

### 4. Запросить итоги
AskUserQuestion (3 вопроса, можно одним опросом):
1. Что сделано (свободный текст; «пропустить» → автогенерация из выполненных пунктов).
2. Неожиданные сложности (или «нет»).
3. Реально заняло времени.

### 5. Записать «Итоговый отчёт» в спринт-документ
Найти раздел `## Итоговый отчёт`; если его нет — добавить в конец. Через Edit заполнить:
```markdown
## Итоговый отчёт
**Дата:** {YYYY-MM-DD} · **Итог:** {X}/{X+Y} ({Z}%) · **Время:** {из шага 4}

### Выполнено
{список - [x]}
### Не выполнено
{список - [ ] или «— всё выполнено»}
### Синтез
{текст пользователя или автогенерация}
### Уроки
{что сработало / сложности / что улучшить — если указано}
```

### 6. Статус
Через Edit в шапке: `Статус:` → `✅ Готов` (+ при наличии добавить `Дата окончания: {YYYY-MM-DD}`).

### 7. Синк ROADMAP / индекса
- `docs/ROADMAP.md`: Grep ссылку/строку спринта → пометить ✅ (дата). Если нет — предупредить.
- `docs/SPRINTS.md`: отметить спринт ✅ в индекс-таблице.

### 8. Уроки в memory (предложение)
Если в шаге 4 есть важное — AskUserQuestion: «Записать урок в memory (`feedback`/`project`)?». Если да — создать файл в memory-папке проекта (формат из глобального CLAUDE.md: frontmatter name/description/type, тело с **Why:**/**How to apply:**) и добавить строку в `MEMORY.md`. Без согласия — НЕ писать.

### 9. Коммит/PR (предложение)
**Сначала прогнать `pnpm verify`** (полный гейт: typecheck+lint+format:check+test+build) — финализировать спринт можно только при зелёном; если красный — отказать и сообщить, что чинить.
AskUserQuestion: «Закоммитить финализацию?». Если да — подсказать (на текущей ветке):
```bash
git add docs/ && git commit -m "docs(sprint-{N}): финализация — <кратко>"
```
PR — обычным `gh pr create`, если есть GitHub-remote (forgejo/`/dispatch` не используются).

### 10. Финал
```
✅ S{N} финализирован · {X}/{X+Y} ({Z}%)
🗺 ROADMAP/индекс обновлены
💾 Memory: <урок записан / нет>
📦 Коммит: <сделан / отложен>
```
**Архивирование (по умолчанию):** перенести завершённый спринт-документ в `docs/sprints/archive/` через `git mv` (сохраняет историю) и обновить ссылку в индексе `docs/SPRINTS.md` (и в ROADMAP, если есть) на путь `sprints/archive/...`. Активные спринты остаются в корне `docs/sprints/`.

## Anti-patterns
1. Не финализируй с невыполненными без явного согласия.
2. Не пиши в memory без согласия пользователя.
3. Не используй `grep -rl`/`grep -oP` на Windows — Glob + Grep отдельно.
4. Не трогай ROADMAP/индекс «втихую» — иначе спринт «висит» в документации.

## Быстрый режим
`--quick` — шаги 1–6 (без ROADMAP/индекс-синка, memory, коммита).
