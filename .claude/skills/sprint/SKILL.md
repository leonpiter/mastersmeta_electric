# Статус спринта

Read-only. Показывает статус спринта проекта **see_elecctrical_copy**: метаданные, прогресс по чеклисту, ближайшие и выполненные задачи.

Конвенции проекта: спринты — `docs/sprints/S{N}-{slug}.md` (см. индекс `docs/SPRINTS.md`). Статусы: `📝 Запланирован` / `🚧 В работе` / `✅ Готов` / `📝 Опционально` (детектируй и по эмодзи, и по словам «запланирован/в работе/готов/опционально»).

## Шаги

### 1. Найти файл спринта
- Если задан `$ARGUMENTS` (номер) — Glob `docs/sprints/S*.md`, выбери файл, чей номер (regex `S0*<N>-`) совпадает.
- Если не задан — Glob `docs/sprints/S*.md` + Grep статуса `🚧|В работе`; если активных несколько — покажи список и спроси. Если активных нет — покажи последний завершённый и предложи следующий запланированный.

Прочитай файл.

### 2. Метаданные (из шапки)
Веха, Статус, Зависит от, Ветка, Оценка (если есть).

### 3. Прогресс по чеклисту
```bash
grep -c '\- \[x\]' <файл>   # выполнено
grep -c '\- \[ \]' <файл>   # осталось
```
Прогресс % = x/(x+y)×100. Если grep не сработал — посчитай через Read + regex.

### 4. Отчёт
```
📋 S{N} · <название>
Веха: <A–G> · Статус: <статус> · Ветка: <ветка>
Прогресс: x/(x+y) (Z%)  [███████░░░]
Зависит от: <…>

Ближайшие (топ-5 невыполненных):
- [ ] …

Выполнено (последние 5):
- [x] …
```
Пустые секции не показывай. Прогресс-бар — ASCII (█/░).
Если Z=100% и статус ≠ ✅ — добавь: «✅ Все задачи выполнены — запусти `/sprint-finalize {N}`».

## Anti-patterns
1. Прогресс — только по `- [x]`/`- [ ]`, не по своему усмотрению.
2. Это read-only скилл — ничего не записывай и не коммить.
