# S31 · Расширение базы УГО по ГОСТ + справочник буквенных кодов

- **Веха:** E — Сообщество / OSS
- **Статус:** 🚧 В работе
- **Зависит от:** S2 (формат `*.symbol.json` / библиотека), S30 (библиотека-папка)
- **Ветка:** main
- **Оценка:** 1–2 недели

## Цель

Довести встроенную библиотеку УГО (сейчас 18 символов / 14 категорий) до полного набора
по ГОСТ для принципиальных электрических схем: УГО на каждый буквенный код, 1/2/3/4-полюсные
варианты коммутационных аппаратов, **точные размеры** из методических документов; плюс
справочник буквенных кодов (ГОСТ 2.710-81) как данные `core` + модалка + страница docs.

## Контекст

Источники (согласовано с пользователем):
- `docs/manual/UGO.pdf` (УлГТУ, Свиридов 2015) — Табл.1 (~80 УГО с размерами в мм), Табл.9 (коды).
- `docs/manual/14_Мошкин-ВИ_2013_МУ_3.pdf` (КГУ, Мошкин 2013) — Рисунок 1 (91 пример) и
  **Таблица А1 «Наименования электрических элементов и их буквенный код»** — канон справочника.

Зона **доменной модели** (`categories.ts`, формат/набор `SymbolDef`) → согласование проведено
в ходе планирования. Решения:
1. **Справочник** = Табл. А1 полностью → `core` (данные) + модалка в UI + страница docs.
2. **Охват библиотеки** — УГО на каждый код Табл. А1 + коды, нарисованные в Табл.1 первого
   документа, но отсутствующие в А1: `SQ`, `SK`, `EL`, `KK`. Одинаковые по ГОСТ начертания
   переиспользуются (все реле = катушка-прямоугольник, все приборы = круг) — отличие в
   `componentCode`/категории. Сигла берётся из `sym.componentCode`
   (`commands.ts:73`, `blocks.ts:100`) → подтипы реле в одной категории «Реле».
3. **Полюсность** — отдельные УГО на полюс (`gost.qf.1p/.2p/.3p/.4p` и т.п.); рендер/связность не трогаем.
4. **Точные размеры — жёсткое требование** (не «на глаз»): каждый УГО строится по размерам
   из документов и **сверяется с образцом** (рендер страницы PDF в PNG) до фиксации.
5. **УГО планов** (ГОСТ 21.614-88) — вне охвата (будущий спринт).
6. **Штампы** (основная надпись ГОСТ 2.104) — отдельный спринт **S32** (Часть B плана).

Размерный справочник (из документов) — критерий приёмки геометрии:

| Элемент | Код | Размеры (мм) |
|---|---|---|
| Катушка эл.-мех. устройства (реле, контактор) | K/KM | прямоугольник **12 × 6** |
| Резистор постоянный | R | **10 × 4** |
| Предохранитель плавкий | FU | **10 × 4**, с осевой линией |
| Рубильник/выключатель НВ | QS | угол **30°**, длина **6** |
| Выключатель высоковольтный | Q | угол **30°**, **2** и **6** |
| Контакт контактора размыкающий | KM | перемычка **5**, размах **7**, смещение **6** |
| Кнопка управления | SB | **7** (толкатель), **6** (зазор) |
| Контакт с замедлением | — | **7** |
| Трансформатор силовой | T | Ø**10**, стрелка **20**, угол **45°**, центры **6**; основные ×2 |
| Трансформатор тока | TA | Ø**10**, радиус дуг **2,5** |
| Трансформатор напряжения | TV | Ø**10**, центры **6** |
| Лампа | EL/HL | Ø **6–8** |
| Прибор измерительный | P* | Ø**10**, квадрат **10×10** |
| Диод | VD | Ø**10**, **60°**, **6**, **2,5** |
| Генератор/машина | G/M | Ø**10**; основные ×2 |
| Реактор | LR | Ø**12** |

Выводы — на сетке **5 мм** (`pin.x/pin.y` кратны 5). План: `.claude/plans/…e-tingly-dolphin.md` (Часть A).
