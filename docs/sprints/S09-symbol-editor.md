# S9 · Редактор символов / блоков

- **Веха:** C — Создание контента
- **Статус:** 📝 Запланирован
- **Зависит от:** S2

## Цель
Пользователь создаёт собственные УГО прямо в приложении **и может править системные (встроенные) УГО**.

## Scope
- [ ] Рисование примитивов (линия / прямоугольник / дуга / окружность); выводы на сетке 5 мм.
- [ ] Выбор поведения `Symbol.kind`: coil / component / component-aux / contact-NO/NC / terminal / black-box.
- [ ] Атрибуты символа; сохранение в `*.symbol.json`.
- [ ] **Две библиотеки УГО:** системная (встроенные ГОСТ, `symbols-gost.ts`) и пользовательская (`*.symbol.json` на диске/в проекте). В панели «Символы» — разделены (напр. группы/бейдж «системный»/«мой»).
- [ ] **Редактирование системного УГО:** открыть встроенный символ в редакторе → правки сохраняются как **override** в пользовательскую библиотеку (системный остаётся эталоном, можно «сбросить к системному»). Для разработчика — режим экспорта правки обратно в `symbols-gost.ts` (встроить в дистрибутив).
- [ ] Привязка к S24: тот же редактор даёт вкладку «Footprint (1:1)» к символу/типу.

## Технический дизайн
- `SymbolLibrary` поддерживает источники: built-in (GOST) + user (override по `id`). Резолв: user перекрывает built-in.
- «Сброс к системному» = удалить override. «Экспорт в систему» (dev) = записать в `symbols-gost.ts` (или в seed-пак).

## DoD
- [ ] рисуешь символ, задаёшь выводы/поведение, сохраняешь в пользовательскую библиотеку; он доступен для вставки
- [ ] системный УГО открывается в редакторе и правится (override); «сброс к системному» возвращает эталон
