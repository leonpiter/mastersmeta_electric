# S9 · Редактор символов / блоков

- **Веха:** C — Создание контента
- **Статус:** 🚧 В работе (редактор MVP готов) · ветка `phase-2-symbols`
- **Зависит от:** S2

## Цель
Пользователь создаёт собственные УГО прямо в приложении **и может править системные (встроенные) УГО**.

## Scope
- [x] Рисование примитивов (линия / прямоугольник / окружность); выводы (snap 5 мм, графика 1 мм). Дуга — позже.
- [x] Выбор поведения `Symbol.kind` (coil / component / component-aux / contact-NO/NC / terminal / connector / black-box) + атрибуты (название/код/категория). Валидация через `validateSymbol`.
- [x] Сохранение в **пользовательскую библиотеку** (localStorage; `apps/desktop/src/user-symbols.ts`). Формат `*.symbol.json` на диске/в проекте — позже (S12).
- [x] **Две библиотеки УГО:** системная (GOST) + пользовательская; в панели «Символы» бейдж «мой»; пользовательский перекрывает системный по `id` (`SymbolLibrary.add`/`remove`).
- [x] **Редактирование системного УГО:** открыть в редакторе → сохраняется как **override** (тот же `id`); «Сбросить к системному» удаляет override и восстанавливает эталон из `symbols-gost.ts`. «Дублировать» — новый id. «Удалить» — для пользовательских.
- [ ] Экспорт правки обратно в `symbols-gost.ts` (dev) — позже.
- [ ] Привязка к S24: вкладка «Footprint (1:1)» — позже.

## Технический дизайн
- `SymbolLibrary` поддерживает источники: built-in (GOST) + user (override по `id`). Резолв: user перекрывает built-in.
- «Сброс к системному» = удалить override. «Экспорт в систему» (dev) = записать в `symbols-gost.ts` (или в seed-пак).

## DoD

- [x] системный УГО открывается в редакторе и правится (override); «сброс к системному» возвращает эталон
- [x] визуальная приёмка через Playwright (рисунок прямоугольника + 2 выводов, сохранение, бейдж «мой»)
