# S6 · Каталог изделий

- **Веха:** B — Данные и отчёты
- **Статус:** ✅ Готов · Дата окончания: 2026-05-26
- **Зависит от:** S2, S5

## Цель
Привязка изделий производителей к элементам (для спецификации).

## Scope
- [x] MVP-каталог: `core/catalog.ts` (`CatalogPart` {код, производитель, тип, номинал, описание, ГОСТ-код}, класс `Catalog`, `partLabel`) + встроенный набор `catalog-builtin.ts` (автоматы/контакторы/реле/… IEK/Schneider/DKC).
- [x] Привязка изделия к инстансу: `SymbolInstance.catalogCode` + `EditInstanceCommand`; в свойствах элемента — **двухуровневый выбор Производитель → Изделие** (`Catalog.manufacturers` / `byCodeAndManufacturer`), отфильтрованный по ГОСТ-коду. Артикул устройства = код master-инстанса (`DeviceInfo.catalogCode`); в списке «Контакты» виден артикул.
- [ ] Открытый формат `*.part.json` + JSON Schema; импорт каталога из файла; браузер каталога (производитель→группа→изделие) — позже (S6+/S12).
- [ ] **Несколько кодов** через `;` (составное изделие); конфигурация контактов из кода (наполняет contact cross) — позже.

## DoD

### Выполнено
`core/catalog.ts` (`CatalogPart`, `Catalog`, `partLabel`, `manufacturers`/`byCodeAndManufacturer`) + встроенный мини-каталог (IEK/Schneider/ABB/DKC); `SymbolInstance.catalogCode` + `EditInstanceCommand`; в свойствах элемента **двухуровневый выбор Производитель → Изделие**; артикул устройства (`DeviceInfo.catalogCode`) виден в списке «Контакты».
