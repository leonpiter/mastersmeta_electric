## Цель
Автоматическая нумерация проводов по цепям (ГОСТ 2.709).

## Scope
- [x] Ядро: `Wire.number/locked`; `AutoNumberCommand` (по цепям) + `SetWireNumberCommand` (ручной/потенциал/lock). +тесты.
- [x] Именованные потенциалы L1/L2/L3/N/PE (ручной номер + lock; в свойствах провода кнопки-пресеты).
- [x] UI: меню «Нумерация» (Сгенерировать… / Очистить всё / Очистить выбранное) + диалог опций (метод/старт/шаг/перенумеровать заблокированные); номера видны на проводах (со смещением перпендикулярно — над горизонтальным, слева от вертикального).
- [ ] Нумерация цепей: сквозная / по номеру листа (формат `Page.Number`) — позже.
- [x] Типы проводов (силовой / управление) — есть (S3); видимость номера.

## Эталон UI (See Electrical «Wire Numbers»)
Меню Numbers: **Generate… / Clear All / Clear Selected**. Диалог:
- **Метод:** Potential numbering (одна цепь = один номер; + «Skip terminals») · Wire numbering (каждый провод уникален) · между компонентами разных Function/Location · порядок по соединениям.
- **Формат номера потенциала:** использовать имена потенциалов как номера; Startnumber; Step.
- **Формат номера провода:** Number · Page.Number · Page/Cell/Number — разделитель, Startnumber, Step.
- **Перенумеровать:** Unlocked wires · Locked wires (снимая блокировку).

MVP сейчас: метод potential/unique, Start/Step, «перенумеровать заблокированные», очистка всё/выбранное. Формат `Page.Number` и «Skip terminals» — после S8 (клеммы) / мультилиста.
