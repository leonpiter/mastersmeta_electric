## Цель
УГО охватывают весь спектр: мелкие контакты/кнопки → крупные приборы (контроллеры/ПЛК,
реле времени, аппараты) → готовые блоки управления со своими клеммниками. Доменная модель
`SymbolDef` уже поддерживает символы любого размера/числа выводов — узкое место в редакторе.

## Технический дизайн
- Крупные символы = апгрейд редактора: `panX/panY/zoom`, `scalePx=PX_PER_MM*zoom`, content `<g>` transform, сетка-`<pattern>`, `fit()` через `symbolBounds`.
- Блоки — «макрос»: выделенные инстансы → шаблон относительных размещений; вставка строит обычные инстансы (одна `MacroCommand`/`InsertBlockCommand`), последовательная авто-перенумерация. Плоская модель и master/slave/отчёты работают без изменений.
- Зеркало контактов: `SymbolDef.contacts?: {pins; column:"M"|"NO"|"NC"}[]`; `coilContactRows(device, coilSym)` сливает комплект с размещёнными (адрес «лист.зона»).

### Уроки
- **Что сработало:** инкремент по фазам с Playwright-приёмкой каждой; чистые функции в ядре (`instanceLabels`, `coilContactRows`, `captureBlock`) — тестируемы и переиспользуемы рендером.
- **Сложности:** мультивставка блока требует *последовательной* нумерации — наивная конструкция всех команд до выполнения даёт одинаковые сиглы; решено бегущим списком `existing` в `InsertBlockCommand`.
- **На будущее:** пустая пользовательская категория не видна в библиотеке (панель группирует по символам) — норма для текущего флоу (категория создаётся при сохранении символа), но учесть при экспорте паков (S12).
