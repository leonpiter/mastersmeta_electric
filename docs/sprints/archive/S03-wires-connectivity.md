## Цель
Движок цепей (`Net`): связность вычисляется из геометрии.

## Scope
- [x] `Wire` (≠ `Line`) + `Page.wires`; рисование **1-Wire** (полилиния кликами, snap к сетке, цепочка, Esc — завершить) + слой проводов; `AddWireCommand`/`RemoveWireCommand` (обратимы).
- [x] **union-find**: `computeNets` (точки = концы/точки проводов + выводы символов `instancePins`) — `Net` не хранится, пересчитывается (принцип 2). +4 теста (4 топологии).
- [x] Детект висящих выводов (`danglingPins`).
- [x] Вкладка «Соединения» в ленте + инструмент «Провод».
- [x] 3-полюсный провод (3 фазы, шаг 5 мм) + авто-точки узлов (`computeJunctions`: жирная точка на Т-ответвлении/схождении ≥3, без точки на простом пересечении/угле).
- [x] Выделение/удаление провода + свойства (тип/сечение/цвет, `EditWireCommand`).
- [x] Подсветка цепи (по `Net`) при выборе провода: вся электрически связанная группа проводов + кольца на выводах.
- [x] Разрез провода при вставке символа на него (`SplitWireCommand`, принцип 2); авто-реконнект концов проводов при перемещении символа (`MacroCommand` + `MoveWireEndpointCommand`).

## Технический дизайн
- Сбор точек (pins + концы проводов + junction) → объединение по совпадению (после snap) → компоненты связности (`packages/core/src/connectivity.ts`).
- `Net` не хранится — пересчитывается (CLAUDE принцип 2).
