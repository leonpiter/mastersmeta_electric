# S29 · Адресные соединители страниц (перенос провода между листами)

- **Веха:** A — Редактор схем (расширяет S3 связность / S4 нумерацию / S5 кросс-рефы)
- **Статус:** ✅ Готов · ветка `sprint-29-wire-crossref` (влита в `main`) · Дата окончания: 2026-05-27
- **Зависит от:** S3 (Wire/Net), S4 (нумерация цепей), S5 (зонная адресация), S26 (мультилист/навигация)
- **Ветка:** sprint-29-wire-crossref
- **Оценка:** 2–3 дня

## Цель
УГО «соединитель страниц» (off-page connector): когда провод заканчивается на одном листе и
продолжается на другом, на концах ставятся адресные стрелки, которые **электрически связаны**
(одна цепь, один номер провода) и показывают адрес партнёра (`лист·зона`). Одиночный клик по
стрелке переносит на лист партнёра с подсветкой.

## Контекст
Согласовано с пользователем (зона доменной модели + движка связности, принцип 2):
- **Связь по общей метке сигнала.** У соединителя — метка; все соединители с одной меткой =
  одна цепь (union их локальных нетов через листы). Поддерживает >2 точек продолжения.
- **Метка по умолчанию — авто по номеру цепи** (S4), с ручным переопределением.
- **Адрес** партнёра — `лист N · зона B3` (зоны уже есть: `zoneGrid` в `sheet.ts`; формат зоны как в S5 кросс-рефах).
- **Постановка** — двойной ЛКМ в конце провода ставит соединитель в точку конца.
- **Навигация** — одиночный клик по соединителю → лист партнёра + подсветка (при >1 партнёре — список).

Смежное: символы (новый kind), нумерация (наследование номера), отчёты (таблица соединений
должна видеть сквозную цепь), сериализация (`signal` на инстансе → версия схемы/миграция не
требуется, поле опциональное).

## Технический дизайн
- **core:**
  - `symbol.ts`: `SymbolKind` += `"page-connector"`; `model.ts`: `SymbolInstance.signal?: string`.
  - `connectivity.ts`: `computeProjectNets(pages: Page[], library)` — для каждого листа считаем локальные неты (как сейчас), затем union-find по группам соединителей с одинаковым `signal` (берём локальный нет вывода соединителя). Возвращает глобальные неты с привязкой к листам.
  - `crossref.ts` (нов.): `zoneOf(page, x, y)` → `{ sheet, zone }`; `connectorPartners(pages, signal)` → адреса прочих соединителей; `autoSignal(net)` → метка из номера цепи.
  - `numbering.ts`: номер цепи распространяется на глобальный нет (через `computeProjectNets`).
- **desktop:**
  - `symbols-gost.ts`: `gost.page-connector` — стрелка (4 направления через rotation), 1 вывод в основании.
  - `canvas.ts`: двойной ЛКМ при черчении провода → завершить + поставить соединитель; рендер метки+адреса под стрелкой; одиночный клик → навигация (через `PageTabs`/переключение листа + флеш-подсветка).
  - команда `AddPageConnectorCommand` (обратимая), правка метки через существующий `EditInstanceCommand`.

## Открытые вопросы
- Направление стрелки: авто по направлению провода или явный выбор? (старт — авто + поворот вручную).
- Формат адреса: `2·B3` vs `=2/B3` — уточнить по ГОСТ-оформлению при первой визуальной приёмке.
