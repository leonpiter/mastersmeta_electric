# S26 · Рабочее пространство: несколько проектов + лента страниц

- **Веха:** Сквозная (UI-оболочка), расширяет S21
- **Статус:** 🚧 В работе · ветка `phase-2-symbols`
- **Зависит от:** S21 (оболочка), S2

## Цель
Открыть несколько проектов одновременно (каждый — своё дерево в левом доке «Структура»)
и работать со страницами как с вкладками: **лента открытых страниц** под лентой инструментов,
над канвасом. Можно держать открытыми несколько листов, в т.ч. из разных проектов.

## Scope
- [x] Левый док: несколько проектов (каждый — отдельное дерево Проект → Схема → Листы); активный лист подсвечен. «Создать»/«Открыть» **добавляют** проект. ПКМ корня → «Закрыть проект» (последний не закрывается).
- [x] Лента открытых страниц (`#pagetabs`) под тулбаром, над канвасом: вкладка = «проект / Лист N», клик — активировать, × — закрыть (последнюю не закрываем); открытие/создание листа добавляет вкладку.
- [x] Активная страница определяет канвас и активный проект (отчёты/устройства/каталог/сохранение работают с активным).
- [x] Команды undo/redo — общий стек; `stack.subscribe` реконсилит вкладки/активную страницу после add/remove/undo. (Стек на проект — позже.)
- [x] **Наименование листа**: ПКМ листа → «Наименование листа…» (`SetPageTitleCommand`, обратимо) → видно в дереве («Лист N · A3 · <название>») и в штампе (графа 1, ГОСТ 2.104).

## Технический дизайн
- App-уровень: `projects: Project[]` (открытые), активный `project` (let), `openPages: Page[]` (вкладки),
  активная страница. `projectOf(page)` ищет владельца. Активация листа: `project = владелец;
  project.activePageId = page.id; view.setPage(page)`.
- `core` не меняется (Project уже самодостаточен; workspace — слой приложения).

## DoD
- [x] открыто ≥2 проектов в левом доке; листы из разных проектов открываются вкладками; переключение вкладок меняет канвас и активный проект
- [x] «Создать»/«Открыть» добавляют проект; закрытие вкладки/проекта не ломает состояние (последние не закрываются)
- [x] `pnpm verify` зелёный; визуальная приёмка через Playwright (2 дерева, 3 вкладки, переключение/закрытие)
