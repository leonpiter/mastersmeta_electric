## Технический дизайн
- `Symbol` рендерится из JSON в SVG (примитивы + выводы); превью в панели (`apps/desktop/src/symbol-render.ts`).
- Поворот/зеркало — SVG-трансформ инстанса `translate · rotate · scale(±1)`, совпадает с `transformLocalPoint` в ядре → положения выводов считаются детерминированно (задел под связность S3).
- Геометрия УГО приближённая (как штамп S1); уточнение начертаний по ГОСТ 2.721/2.755 — отдельная задача.

## Прогресс — инкремент 1 (2026-05-25)
**Решения (ведущий разработчик, по делегированию пользователя):**
- **Валидация формата** — структурная `validateSymbol()` в ядре + канонический JSON Schema-документ; подключение ajv в CI отложено в S12 (там это явный scope) → ядро остаётся zero-dependency.
- **`SymbolInstance` без `Device`** — в S2 один инстанс = одно логическое устройство (1:1); поле `deviceId` зарезервировано. Полный слой `Device`/каталог — S5/S6 (YAGNI сейчас).
- **Автонумерация сканированием** (а не хранимым счётчиком) — устойчива к undo/redo и сериализации.
- **Стартовый набор 7/11** — движок важнее количества; оставшиеся 4 УГО — чистые данные, добавляются тривиально.

**Проверено:** `core` typecheck + 26 тестов (14 новых: трансформы, габариты, автонумерация, валидация, команды); `desktop` typecheck + build зелёные.

## Прогресс — инкремент 3: оформление под See Electrical (2026-05-25)
