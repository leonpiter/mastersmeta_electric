## Цель

Привести основную надпись (штамп) к ГОСТ 2.104 **Форма 1**: точная размерная геометрия,
редактирование всех полей через диалог, настраиваемый набор ролей, авто-нумерация листов.

## Контекст

Штамп уже есть (S1), но геометрия приближённая и из UI редактируется только наименование:
- **Модель** `packages/core/src/sheet.ts`: `TitleBlock { title, designation, company, letter,
  mass, scale, sheet, sheetsTotal, developer, checker }`, `TITLE_BLOCK_SIZE = 185×55`,
  `defaultTitleBlock()`; форматы `FORMATS` (A3 420×297, A4 297×210, только альбом);
  `frameRect()`, `zoneGrid()`.
- **Рендер** `apps/desktop/src/canvas.ts`: `renderSheet()` + `renderTitleBlock()` (185×55,
  геометрия **хардкод/приближённая**; роли Разраб./Пров./Т.контр./Н.контр./Утв.).
- **UI**: редактируется только `title` (диалог `sheet-name-dialog`, `SetPageTitleCommand`).
  Остальные поля рисуются из модели, но **диалогов нет**; авто-нумерации листов нет.
- **Экспорт**: печать → PDF (одно-/многостраничный), штамп включается. Тесты `sheet.test.ts`
  (рамка/зоны/размер штампа).

Образцы форм (оба документа `docs/manual/`): УлГТУ Рис. П5 (Форма 1) и КГУ Табл. Б2.

**Решения** (согласовано):
1. Охват — **только Форма 1** (185×55). Формы 2/2б (текстовые/перечень) — вне охвата (будущее).
2. Роли — **настраиваемые (data-driven)**: список `{role, name}`; дефолт — производственная
   ГОСТ 2.104 (Разраб./Пров./Т.контр./Н.контр./Утв.), пользователь может изменить.
3. Размеры Формы 1 — **точные**, как данные (не хардкод в рендере).

Размерная сетка Формы 1 (из образцов):
- Габарит **185 × 55**. Левый блок: колонки **7, 10, 23, 15, 10** (=65), 5 строк ролей × 11.
  Шапка «Изм | Лист | № докум. | Подп. | Дата».
- Центр — наименование (проект/чертёж), блок шириной 120.
- Правые ячейки: **Литер** (15) / **Масса** (17) / **Масштаб** (18) сверху;
  **Лист** (20) / **Листов** (30) снизу. Обозначение «…-…-….Э3» — верхняя строка.

План: `.claude/plans/…e-tingly-dolphin.md` (Часть B).
