## Definition of Done (для каждого спринта)

- ✅ `pnpm test` зелёный (юнит-тесты на `core`; для рендера — snapshot SVG).
- ✅ `pnpm typecheck` и `pnpm build` без ошибок.
- ✅ Видимый результат в `pnpm dev` (визуальная приёмка пользователем).
- ✅ Документы обновлены (ARCHITECTURE/REFERENCE при изменении модели).
- ✅ Соблюдены принципы CLAUDE.md (headless-ядро, Wire≠Line, uuid, сериализуемые команды).

## Веха A — Редактор схем

### S0 · Каркас ✅ ГОТОВ
Монорепо (core + desktop), канвас: pan/zoom, сетка, привязка, command-стек undo/redo, 8 тестов.

### S1 · Лист и штамп
**Цель:** корректный лист по ГОСТ. **Scope:** форматы A3/A4, рамка (ГОСТ 2.301: поля 20/5/5/5), зонная
сетка (1…N / A…M), основная надпись (ГОСТ 2.104, шаблонная, data-bound), экспорт листа в PDF.
**DoD:** печатается/экспортируется лист со штампом и рамкой; поля штампа заполняются из данных проекта.

### S2 · Символы и расстановка
**Цель:** библиотека УГО и вставка. **Scope:** формат `*.symbol.json` + JSON Schema; панель библиотеки
(категории, фильтр, favorites); вставка кликом/drag-drop, поворот/зеркало, привязка выводов к сетке 5 мм;
авто-позобозначение по ГОСТ 2.710; стартовый набор: QF, QS, УЗО/дифавтомат, KM, K, FU, SB, HL, M, T, XT.
**DoD:** расставляешь элементы с автоматическими позобозначениями, поворот/зеркало работают.

### S3 · Провода и связность
**Цель:** движок цепей. **Scope:** `Wire` (≠ `Line`), рисование 1-Wire/3-Wire, junction, разрез провода
при вставке символа, **union-find** (Net/потенциалы), авто-реконнект при перемещении, подсветка цепи.
**DoD:** цепи вычисляются из геометрии; перемещение элемента тянет провода; подсветка потенциала.

### S4 · Автонумерация и потенциалы
**Цель:** номера проводов. **Scope:** именованные потенциалы (L1/L2/L3/N/PE, верх/низ), нумерация цепей
(ГОСТ 2.709), типы проводов (силовой/управление), ручной номер + блокировка (`locked`), видимость номера.
**DoD:** провода нумеруются автоматически по цепям; ручной override и блокировка работают.

### S5 · Master/slave и кросс-референсы
**Цель:** связь катушка↔контакты. **Scope:** `Device` vs `SymbolInstance`; привязка контактов к катушке по
общему имени + номера выводов (A1/A2, 13/14, 21/22); авто **contact cross** (зеркало контактов); кросс-рефы
между листами + зонная адресация (`→3.1`); «список контактов» (поиск несвязанных).
**DoD:** контакт ссылается на катушку, у катушки авто-зеркало контактов с адресами.

## Веха B — Данные и отчёты

### S6 · Каталог изделий
**Цель:** артикулы для спецификации. **Scope:** формат `*.part.json` + Schema; панель «символы по коду»
(производитель→группа→изделие); привязка `Device`→`CatalogPart` (**несколько кодов** через `;`); конфигурация
контактов из кода (наполняет contact cross); расширяемые каталоги.
**DoD:** элементу присваивается артикул; контакты подтягиваются из кода каталога.

### S7 · Генерация отчётов
**Цель:** документы из схемы. **Scope:** перечень элементов (ГОСТ-оформление), таблица соединений; «живые»
списки + генерация листов-отчётов; экспорт PDF/Excel с **round-trip** правкой кодов; индекс проекта.
**DoD:** из схемы получаем перечень элементов и таблицу соединений, экспорт в PDF/Excel.

### S8 · Клеммники и разъёмы
**Цель:** XT и разъёмы. **Scope:** `Terminal`/`Connector` (number/sorting/pins), клеммник XT; генерация
(табличная/графическая/многоуровневая); `Wire2Terminal`; пины разъёма из каталога.
**DoD:** клеммник и разъёмы с раскладкой и таблицей соединений.

## Веха C — Создание контента

### S9 · Редактор символов / блоков
**Цель:** свои УГО. **Scope:** рисование примитивов (линия/прямоуг./дуга/окружность); выводы на сетке 5 мм;
выбор поведения `Symbol.kind` (coil / component / component-aux / contact-NO/NC / terminal / black-box);
атрибуты символа; сохранение в `*.symbol.json` (библиотека пользователя).
**DoD:** пользователь рисует символ, задаёт выводы/поведение и сохраняет в библиотеку.

### S10 · Компоновка шкафа
**Цель:** физический вид. **Scope:** модуль «забудова» (масштаб 1:10), шкаф/рейки/кабель-каналы; «список
символов для вставки» (связь схема→шкаф через `Device`); габариты footprint из каталога (X/Y/Z); размеры;
индекс-таблица материалов; слои.
**DoD:** физическая компоновка шкафа, связанная со схемой; индекс-таблица.

## Веха D — Поставка

### S11 · Оболочка + авто-обновление
**Цель:** инсталлятор. **Scope:** **выбор Tauri vs Electron**; `Platform`-порт (диалоги/файлы); сохранение/
открытие `.seeproj` (zip + версия схемы + миграции); GitHub Actions: сборка/подпись/публикация + манифест;
**ключ updater + версия схемы — с первого релиза**; уведомление о новой версии + авто-обновление.
**DoD:** подписанный `.exe` ставится; при наличии интернета сам находит и накатывает обновления.

## Веха E — Сообщество / OSS

### S12 · Открытые форматы и паки
**Scope:** JSON Schema для `*.symbol.json`/`*.part.json` + валидация в CI; библиотечные паки (manifest +
версия + лицензия); установка/экспорт паков; решение по лицензиям (код MIT/Apache, контент CC-BY).
**DoD:** пак валидируется в CI и устанавливается в приложение.

### S13 · Community-реестр (статический)
**Scope:** публикация/поиск/установка паков с GitHub/CDN (без сервера). **DoD:** пак публикуется и ставится
из реестра внутри приложения.

## Веха F — LLM-генерация

### S14 · Core API (низкоуровневые операции)
**Scope:** типизированные методы (`addComponent`, `connect`, `numberWires`, `generateBOM`, `renderSheet`);
headless-рендер SVG/PNG для само-верификации агента. **DoD:** схема собирается кодом и рендерится без GUI.

### S15 · Skills (предметные инструменты)
**Scope:** высокоуровневые операции (`addBreaker`, `addRCD`, `makeBusbar`, `dropToTerminals`).
**DoD:** LLM собирает типовой щит вызовами скиллов, результат валиден.

### S16 · Declarative DSL + авто-раскладка
**Scope:** декларативная спека щита → компилятор раскладки/соединения/нумерации (прецедент — auto-diagramming
из Excel в See Electrical). **DoD:** из описания генерируется корректная многолистовая схема.
